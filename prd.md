# 作業提醒管理系統 PRD (Product Requirements Document)

## 1. 產品概述
**產品名稱**：作業提醒管理系統
**產品簡介**：一款專為大學生設計的應用程式，旨在協助使用者集中管理多門課程的作業、報告與考試時間，透過多管道的定時提醒通知，減少忘記截止日期的情況，並幫助使用者提早規劃時間，避免臨時抱佛腳。

## 2. 目標受眾
- **主要使用者**：大學生
- **使用者痛點**：
  - 課程繁多，難以記住每門課的作業、報告和考試時間。
  - 缺乏有效的提醒機制，經常在截止日期前才匆忙趕工。
  - 忘了是否已經繳交作業，導致重複確認或遺漏。

## 3. 產品目標
- **目標一**：提供作業、考試與報告的集中管理功能。
- **目標二**：透過提醒通知，降低忘記截止日期的情況。
- **目標三**：幫助使用者提早規劃時間，避免臨時抱佛腳。
- **目標四**：簡單好操作的頁面，提升使用便利性。

## 4. 核心功能需求

### 4.1 截止日期追蹤與定時任務
- **手動新增與編輯**：使用者可以手動新增作業、報告或考試，並設定名稱、課程名稱、截止日期與時間、重要程度、備註等。
- **自動抓取 (進階功能)**：提供整合常見學習管理系統 (LMS, 如 Moodle, Canvas) 的功能，自動匯入作業資訊。
- **提醒設定**：使用者可以自訂多個提醒時間 (例如：截止前 1 天、前 3 天、前 1 週，或自訂具體時間點)。系統會根據設定，透過背景定時任務自動觸發提醒。

### 4.2 即時同步繳交狀態
- **狀態切換**：每項任務具備「未完成」與「已完成」狀態，使用者可一鍵勾選完成。
- **停止提醒**：當任務狀態更新為「已完成」時，系統需提供檢查機制，並立即停止針對該項任務的所有後續未發送的提醒通知。

### 4.3 多方通訊軟體即時推送服務
- **跨平台通知**：整合外部 API，確保提醒訊息能直接送達學生的個人行動裝置。
- **支援平台**：
  - **LINE**：透過 LINE Notify 或 LINE Messaging API 發送即時訊息至使用者的 LINE 帳號。
  - **Email (Gmail)**：發送提醒信件至使用者的電子信箱。

## 5. 非功能需求
- **使用者介面 (UI/UX)**：介面需簡潔、直觀，減少學習成本，符合大學生使用習慣。採用現代化、RWD (響應式) 設計，確保在手機和電腦上均有良好體驗。
- **系統穩定性**：定時任務 (Cron jobs) 必須準確無誤地觸發，確保不漏發通知。
- **效能**：頁面載入速度快，狀態更新需即時生效，不延遲。
- **隱私與資訊安全**：妥善保護使用者的聯絡資訊 (Email, LINE 帳號) 與個人課表資料，確保資料傳輸與儲存的安全。

## 6. 系統架構概念 (建議)
- **前端**：React / Next.js 或 Vue / Nuxt.js (網頁版)，提供流暢且動態的使用者操作介面。
- **後端**：Node.js (Express/NestJS) 或 Python (FastAPI/Django)，處理 API 請求、業務邏輯與定時任務調度 (例如使用 node-cron 或 Redis/Celery 做 Message Queue)。
- **資料庫**：PostgreSQL 或 MongoDB，儲存使用者資料、任務列表、提醒設定與狀態。
- **外部整合**：LINE Messaging API, SMTP/SendGrid (Email)。
